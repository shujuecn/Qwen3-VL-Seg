## 使用说明

本文解释 `materials/2605.07141v1.pdf`，论文题目是 Qwen3-VL-Seg。你可以把它理解为：

```text
Qwen3-VL 会根据文字找出目标 bbox
Qwen3-VL-Seg 在这个 bbox 基础上再预测像素级 mask
```

这份导读不复制论文原文，而是按论文顺序做“句意级释义”。每个编号基本对应论文里的一句话，或一组语义紧密的句子。这样既保留原文逻辑，也避免把论文全文搬运出来。

如果你只做过传统 CV，可以先抓住一个主线：

```text
传统检测/分割：类别集合固定，模型输出 box/mask
VL grounding：文字任意，模型根据语言输出 box
referring segmentation：文字任意，模型根据语言输出 mask
Qwen3-VL-Seg：用 MLLM 的语言理解能力找目标，再用轻量 mask decoder 补像素边界
```

## 0. VL 图像任务地图

### 0.1 传统 CV 到 VL 的变化

传统 CV 常见任务：

- 分类：图里是什么类别。
- 检测：图里有哪些固定类别，框在哪里。
- 语义分割：每个像素属于哪个固定类别。
- 实例分割：每个实例一个 mask。

VL 任务把“类别 id”换成了“自然语言”：

- VQA：图像问答，例如问图里有几个人。
- OCR/VQA：读图中文字，再回答问题。
- REC：Referring Expression Comprehension，根据一句话找 bbox。
- RES：Referring Expression Segmentation，根据一句话找 mask。
- open-vocabulary grounding：类别不固定，文字里说什么就找什么。
- open-world referring segmentation：目标类别、描述方式、场景都可能没在训练集中见过。

传统 CV 的标签空间通常是：

```text
COCO 80 类 / Cityscapes 19 类 / 医学某几个结构
```

VL 的标签空间更像：

```text
"左边那个穿红衣服的人"
"桌子下方半遮挡的黑色猫"
"消防车顶部抬起的喷水装置"
```

这就是 Qwen3-VL-Seg 这类工作的意义：不是只识别固定类，而是把语言、空间定位和像素分割连起来。

### 0.2 关键术语

**VLM / MLLM**

Vision-Language Model / Multimodal Large Language Model。可以输入图像和文字，再输出文字、bbox 或其他结构化结果。Qwen3-VL 属于 MLLM。

**vision encoder**

把图像切成 patch/token，提取视觉特征。传统 CV 里可以类比为 backbone，例如 ViT、Swin、ConvNeXt。

**projector / merger**

把视觉 token 转成 LLM 能理解的 embedding 空间。传统 CV 没有这个环节，因为传统模型通常不接语言模型。

**LLM decoder**

语言模型主体。它逐 token 生成文本，也可以生成 JSON 形式的 bbox 坐标。

**visual grounding**

把语言描述落到图像区域上。输出通常是 bbox。

**referring expression**

指代表达。比如“右边穿蓝色衣服的小孩”。它不是类别名，而是一句能唯一指向目标的描述。

**referring segmentation**

给一句指代表达，输出对应目标的 mask。

**SAM**

Segment Anything Model，强大的通用分割模型。很多方法让 MLLM 找 box，再把 box 交给 SAM 出 mask。Qwen3-VL-Seg 的目标是不用外部 SAM。

**mask decoder**

从视觉/语言特征生成 mask 的模块。传统分割里的 decoder/head 就是类似角色。

**box-guided**

用 bbox 引导 mask 生成。直观理解：先大概圈出目标区域，再在圈内细分边界。

**open-world**

不仅类别开放，描述方式和场景也开放。它比 open-vocabulary 更强调真实世界长尾和 OOD。

**ID / OOD**

ID 是训练分布内；OOD 是训练分布外，例如新类别、极端大小、遮挡、夜间、医疗/自动驾驶等风险场景。

**mIoU / cIoU / P@t**

- mIoU：每个样本算 IoU，再平均。
- cIoU：全数据集像素交并比，先累计交集和并集再相除。
- P@t：IoU 超过阈值 t 的样本比例。P@0.9 对边界很敏感。

## 1. 摘要逐句解释

### A01

论文先定义任务：open-world referring segmentation 要把任意自然语言描述定位到图像中的精确像素区域。

传统 CV 对照：这不是 COCO 类别分割，而是“用户说什么就分什么”。用户可以说类别、属性、位置、关系，甚至很长的描述。

### A02

论文说现有 MLLM 已经很会 open-world grounding，但输出通常还是 sparse bbox。

传统 CV 对照：Qwen3-VL 这类模型可以像开放词表检测器一样给框，但 bbox 只是粗定位，不能替代 mask。

### A03

论文指出 bbox 不够做 dense prediction。

传统 CV 对照：检测框不能表达舌边缘、头发、细杆、遮挡边界。医学和编辑类任务通常需要 mask。

### A04

已有 MLLM 分割方法有两类：一类预测轮廓坐标，一类调用外部分割模型。

传统 CV 对照：轮廓坐标像 polygon annotation，表达连续边界会有离散误差；调用 SAM 则像二阶段系统，部署成本更高。

### A05

论文提出 Qwen3-VL-Seg：把 MLLM 预测的 box 当作语义和空间先验，再解码成 mask。

核心思想：bbox 不是最终答案，而是 mask decoder 的强条件。

### A06

论文强调 decoder 很轻，只加约 17M 参数。

传统 CV 对照：不是再挂一个大 SAM，而是在 4B MLLM 上加一个很小的分割头。

### A07

论文 decoder 有四个关键模块：多尺度视觉注入、语义-空间 query、box 引导高分辨率融合、mask-aware refinement。

传统 CV 对照：

- 多尺度视觉注入像 FPN。
- query 类似 DETR/Mask2Former 的 object query。
- 高分辨率融合像 U-Net 跳连，但用 bbox gate 限制区域。
- refinement 像 coarse-to-fine mask refinement。

### A08

论文构造 SA1B-ORS 数据集，来自 SA-1B，并分成 CoRS 和 DeRS。

解释：

- CoRS 偏类别，例如“所有狗”。
- DeRS 偏描述，例如“靠左、戴帽子的那个人”。

### A09

论文还构造 ORS-Bench，用来测试 ID 和 OOD。

传统 CV 对照：不仅看常规验证集，还专门看长尾类别、极端大小、遮挡、光照、医疗/自动驾驶等困难场景。

### A10

实验覆盖 referring segmentation、visual grounding 和 ORS-Bench。

解释：论文不只看 mask，也看 bbox grounding 是否提升，以及通用多模态能力有没有被破坏。

### A11

论文结论是：这种 box-guided 方式在语言复杂的指代分割上优势明显。

传统 CV 对照：SAM 很强，但如果目标描述是“第二排左边那个小物体”，语言理解就很关键。

## 2. Introduction 逐句解释

### I01

论文开头说 MLLM 推动了视觉理解边界。

解释：图像模型不再只输出类别和框，而是可以对图像对话、推理、读文字、定位区域。

### I02

很多视觉任务需要把语言概念和图像局部证据关联起来。

传统 CV 对照：以前你训练 `person` 检测器；现在用户说“正在骑车的人”，模型要把语言组合映射到区域。

### I03

这种能力叫 visual grounding。

解释：grounding 就是“语言落地到图像位置”。最常见输出是 bbox。

### I04

论文列举 GPT、Gemini、Qwen-VL 等模型在 grounding 上有进步。

解释：这些 MLLM 已经能根据自然语言找目标，不再只依赖固定类别分类头。

### I05

它们通过大规模 instruction tuning 学会定位任意实体。

传统 CV 对照：不是只用检测数据训练，而是用大量“图像 + 指令 + 答案”的数据，让模型学会听指令。

### I06

但这些模型输出多是 bbox，粒度不够。

解释：bbox 是语言定位的压缩结果，丢掉了边界、孔洞、细长结构和遮挡形状。

### I07

应用如机器人、医学图像、精细编辑需要精确边界。

传统 CV 对照：机器人抓取需要知道物体边界，医学需要分器官/病灶，图像编辑需要准确 alpha/mask。

### I08

所以自然下一步是从 box-level localization 到 pixel-level mask prediction。

解释：这就是从 REC 走向 RES。

### I09

现有路线一：MLLM + 外部分割模型，最常见是 SAM。

工程理解：MLLM 负责理解语言和出点/框，SAM 负责根据点/框出 mask。

### I10

这类方法有效，但有额外参数、依赖和部署复杂度。

传统 CV 对照：两个大模型串联，显存、延迟、环境依赖、错误传递都更复杂。

### I11

另一路线是不依赖外部分割器，给 MLLM 加轻量 dense head 或自回归 mask 表示。

解释：模型自己输出 mask，不调用 SAM。

### I12

直接用文本/坐标表达 mask 会有离散化问题。

传统 CV 对照：用 polygon 或 token 序列表达像素边界，很难像 CNN decoder 那样自然恢复连续形状。

### I13

轻量 dense decoder 如果只依赖低分辨率 latent feature，也难以恢复精细边界。

解释：MLLM 的视觉 token 通常比原图低很多，直接上采样会糊。

### I14

论文说 open-world referring segmentation 仍研究不足。

解释：已有 RES 数据集往往类别和表达有限，不能覆盖真实世界长尾描述。

### I15

传统 RefCOCO 系列多是闭集或相对有限对象空间。

传统 CV 对照：类似在 COCO 风格图像里做“某个人/某狗”的指代，不代表开放世界。

### I16

GRES 和 ReasonSeg 扩展了多目标、无目标和推理型指令。

解释：这让任务更接近真实语言，但规模和开放性仍有限。

### I17

论文提出核心问题：能否不用重型外部分割模型，把 MLLM 的 grounding 能力转成 mask？

这是整篇论文的主问题。

### I18

Qwen3-VL-Seg 的答案是：用预训练 MLLM 的 bbox 作为结构先验。

传统 CV 对照：先检测再分割，但检测框来自 MLLM 的语言理解，而不是固定类别检测器。

### I19

论文强调 bbox 不是终点，而是 mask decoding 的 anchor。

解释：bbox 同时提供“在哪里”和“要分哪个语义对象”的弱但稳定约束。

### I20

decoder 的四个模块共同把 coarse box 变成 fine mask。

工程理解：先把 MLLM 特征变 dense，再构造 query，再加高分辨率图像细节，再 refine。

### I21

论文还构造大规模训练数据 SA1B-ORS。

解释：因为 open-world 分割缺训练数据，他们从 SA-1B 自动构造语言-mask 对。

### I22

SA1B-CoRS 做类别导向样本，一个表达可能对应同类别多个实例。

传统 CV 对照：例如“分割所有车”，这类似语义/实例分割混合。

### I23

SA1B-DeRS 做描述导向样本，用属性、关系或上下文区分目标。

解释：例如“坐在长椅左侧、穿蓝衣服的人”。

### I24

ORS-Bench 分 ID 和 OOD。

解释：ID 看训练风格内表现，OOD 看真实部署风险。

### I25

OOD 包括类别、尺度、指令复杂度、遮挡、光照和风险场景。

传统 CV 对照：这些就是传统模型最容易崩的 domain shift。

### I26

论文最后总结贡献：模型、decoder、数据集、benchmark、实验。

阅读重点：如果你要复现代码，最重要的是 decoder；如果研究开放世界，数据集构造也很重要。

## 3. Related Work 逐句解释

### R01

Referring Expression Segmentation 研究“语言描述指定的目标”的像素分割。

传统 CV 对照：不是给类别 id，而是给一句话。

### R02

经典 RES 多是单表达、单实例。

解释：一句话通常对应一个目标。

### R03

GRES 扩展到多目标和无目标。

工程意义：模型要知道“没有这个目标”或“一句话对应多个实例”。

### R04

ReasonSeg 加入推理密集型指令。

解释：目标不是直接说类别，而是要根据常识/关系推理。

### R05

这些数据集仍受限于词表、规模或人工成本。

传统 CV 对照：闭集数据集的问题在 VL 里仍存在，只是换成语言表达覆盖度问题。

### R06

MLLM region understanding 是另一条线。

解释：模型不仅回答整图问题，还能理解局部区域。

### R07

一些系统用 SAM 等外部分割模型。

优点：mask 质量强。缺点：系统重，语言和 mask 生成分离。

### R08

SAM3 这类是 segmentation foundation model，不是 MLLM 原生结构。

解释：它偏“分割模型能理解概念”，而不是“语言模型内部产生 mask”。

### R09

SAM-free 方法用轻量 head 或统一自回归表示。

解释：这类方法追求模型一体化。

### R10

Qwen3-VL-Seg 的差异是直接使用 MLLM 预测 box 作为 decoder 先验。

传统 CV 对照：不是让 decoder 从整图盲找，而是先给一个语义定位框。

### R11

数据方面，现有数据规模/词表/构造成本限制 open-world 训练。

解释：要训练开放世界，必须有大量“图像、语言、mask”的组合。

### R12

SA1B-ORS 试图把 SA-1B 的 mask 变成带语言的指代分割数据。

工程理解：SA-1B 有 mask，但没有语义和指令；论文用 MLLM、tagger、SAM2 等工具补上语言监督。

## 4. Method 逐句解释

## 4.1 Overview

### M01

open-world referring segmentation 的输入是图像和自然语言，输出是对应区域的 mask。

传统 CV 对照：输入从固定类别变成自由文本。

### M02

MLLM grounding 强，但通常只输出粗框。

解释：Qwen3-VL 的强项是理解“你说的是谁”，弱项是直接给精确像素。

### M03

论文用 box-guided mask decoder 桥接 grounding 和 segmentation。

核心：把 bbox 作为中间状态。

### M04

decoder 先增强中间 ViT 特征，再和 multimodal visual embeddings 形成 dense memory。

传统 CV 对照：类似把 backbone 多层特征和语义特征融合成一个可分割的 feature map。

### M05

接着把语言语义和 box 几何一起构造成 object query。

传统 CV 对照：类似 DETR query，但 query 不是随机学习出来的，而是由文本和 bbox 初始化。

### M06

为了恢复边界，论文用 bbox gate 控制浅层图像特征融合。

解释：浅层图像有纹理和边缘，但也有噪声；box gate 限制只看目标附近。

### M07

最后把第一轮 mask 证据反馈到 query，做第二轮预测。

传统 CV 对照：coarse mask -> refine mask。

### M08

论文把整个 decoder 概括为 coarse-to-fine。

解释：MLLM 负责 coarse grounding，mask decoder 负责 fine boundary。

### M09

MLLM 提供四类信息：多尺度视觉特征、多模态视觉 embedding、mask token 特征、bbox。

对传统 CV 的关键差异：除了图像 feature，还有文本生成过程中的 token feature。

### M10

最终 mask 是 decoder 对这些输入的函数。

工程理解：mask head 不再只是 `F_image -> mask`，而是 `F_image + F_text + bbox + raw image -> mask`。

## 4.2 Multi-scale Spatial Feature Injection

### M11

论文说高层 MLLM 特征太粗，不能精确分边界。

传统 CV 对照：只用 ResNet C5 上采样做分割通常边界差，所以要 FPN/U-Net skip。

### M12

所以从 vision encoder 取中间层特征。

解释：中间层保留更多空间结构，高层保留更多语义。

### M13

每层先用 `1x1 Conv` 投影到 decoder hidden dim。

传统 CV 对照：FPN lateral conv。

### M14

再用 depthwise conv 分支注入局部空间偏置。

解释：ViT token 全局强，但局部卷积对边缘/纹理更自然。

### M15

分支前有一个可学习缩放 `s`，初始化很小。

工程意义：新模块一开始几乎不改变预训练特征，训练更稳。

### M16

多层特征 concat 后卷积融合为 `F_fuse`。

传统 CV 对照：多尺度融合 feature map。

### M17

把多模态视觉 embedding 投影并 reshape 成二维图。

解释：LLM 内部的视觉 token 是 sequence，要变回 H x W 才方便 dense mask。

### M18

最终 memory 是多模态语义、空间增强特征和位置编码之和。

传统 CV 对照：一个既懂语言又保留空间的分割 feature map。

## 4.3 Spatial-Semantic Query Construction

### M19

object query 要同时包含目标语义和空间身份。

解释：只知道“dog”不够；还要知道是左边那只还是右边那只。

### M20

论文显式使用 grounded box 构造 query。

传统 CV 对照：DETR query 通常靠学习；这里 query 有 bbox 条件。

### M21

bbox 用 `(x1, y1, x2, y2)` 表示。

注意：VL 模型常输出归一化或图像坐标，需要统一坐标系。

### M22

论文把位置和尺度用 Fourier encoding 编码。

解释：Fourier PE 类似 NeRF/Transformer 位置编码，让 MLP 更容易表示连续坐标。

### M23

宽高用 log-scale 变换。

传统 CV 对照：检测框回归常对宽高取 log，因为尺度变化跨度大。

### M24

box embedding 经 MLP 后与 mask token feature 相加。

解释：query = 几何位置 + 语言语义。

### M25

query 再进 transformer decoder attend 到 memory。

传统 CV 对照：像 Mask2Former query 从 feature map 中读信息。

### M26

这样 decoder 不用完全靠语言去整图检索目标，而是从一个有空间先验的 query 开始。

工程意义：训练更容易，mask 更稳定。

## 4.4 Box-Guided High-Resolution Pixel Fusion

### M27

论文说高层特征缺边界细节，所以从原图提浅层 CNN 特征。

传统 CV 对照：U-Net encoder 的浅层特征。

### M28

但浅层特征包含很多背景噪声。

解释：边缘、纹理、颜色都很多，不一定属于目标。

### M29

所以用 bbox 构造 soft spatial gate。

传统 CV 对照：ROI mask / attention gate。

### M30

论文把 bbox 放大 15%，容忍定位误差。

工程意义：如果预测框略小，mask 仍能看到边界外一点的信息。

### M31

gate 的边界用 sigmoid，而不是 hard crop。

解释：soft gate 可微，边界有缓冲，不会突然截断。

### M32

论文设 `alpha = 20` 控制边界陡峭程度。

直觉：alpha 越大越接近 hard box，越小越模糊。

### M33

多 bbox 时取 gate 最大值。

解释：一个指令可能对应多个实例，例如“所有人”。

### M34

视觉特征先逐步上采样。

传统 CV 对照：decoder upsampling / PixelShuffle。

### M35

再把上采样特征和 gated shallow feature 融合。

核心：语义来自高层，边界来自浅层，bbox 限制背景。

## 4.5 Iterative Mask-Aware Query Refinement

### M36

论文说单轮 mask 预测对细结构和复杂场景不够。

传统 CV 对照：很多分割网络会做 coarse-to-fine、cascade 或 refinement。

### M37

第一轮 query 生成 dynamic kernel 来预测初始 mask。

解释：dynamic kernel 是由 query 生成的卷积核/参数，用目标特定方式分割。

### M38

用初始 soft mask 在 pixel feature 上池化目标区域特征。

传统 CV 对照：用预测 mask 做 attention pooling，得到目标专属表示。

### M39

池化时除以 mask 概率和，并加 epsilon 防止除零。

工程细节：这是 soft average pooling。

### M40

把 target-aware feature 投影后加回 query。

解释：query 看过第一轮 mask 后，带着新的局部证据再预测一次。

### M41

第二轮 query 再预测 mask。

传统 CV 对照：第二阶段修边界、补漏、去误检。

### M42

最终 mask logits 上采样到目标分辨率。

注意：logits 不是二值 mask，推理时还要 sigmoid + threshold。

### M43

论文还用 IoU head 预测 mask 置信度。

传统 CV 对照：类似 Mask R-CNN mask score 或检测置信度，但它估计 mask 质量。

### M44

这个 loop 让 query 和 pixel evidence 显式交互。

核心意义：不是只由语言 token 一次性决定 mask，而是让 mask 反过来更新目标表示。

## 5. 数据集构造逐句解释

## 5.1 SA1B-ORS Overview

### D01

论文为 open-world referring segmentation 构造大规模数据集。

解释：没有足够语言-mask 数据，就很难训练开放世界分割。

### D02

数据来自 200 万张 SA-1B 原始图像。

传统 CV 对照：SA-1B 有大量 mask，但很多 mask 没类别、没语言。

### D03

数据分 CoRS 和 DeRS 两个子集。

解释：一个练类别，一个练描述。

### D04

CoRS 支持 category-consistent supervision。

例子：给类别“person”，目标可能是一个或多个 person。

### D05

DeRS 训练 instance-specific 指代。

例子：多个人里“穿条纹衬衫、站右边的人”。

## 5.2 SA1B-CoRS

### D06

SA-1B mask 是碎片化、无类别、无语言的。

传统 CV 对照：有分割候选，但不知道语义，也不适合作为指令训练样本。

### D07

CoRS 用五步把原始 mask 变成类别指代数据。

五步是：实体蒸馏、粗 mask、细 mask 合并、MLLM 验证、caption 生成。

### D08

实体蒸馏先找图中可靠的可指代实体。

解释：过滤掉不适合说成目标的东西，例如无意义碎片。

### D09

论文用 RAM++ 生成开放词表 tags，再做语义过滤。

传统 CV 对照：先用 tagger 猜图里有什么，再筛掉场景词、颜色词等不适合作为目标的标签。

### D10

保留人、车、动物、器物、衣服、食物、建筑等可指代语义。

解释：这些词能对应视觉实体。

### D11

再用 open-vocabulary grounding model 验证实体。

工程意义：tag 说有“boat”，还要确认图里真能定位到 boat。

### D12

用层级 NMS 保留更具体标签，也允许空间重叠类别共存。

传统 CV 对照：避免 “animal/dog” 重复冲突，同时允许 “person/clothing” 重叠。

### D13

粗 mask 获取阶段，Qwen3-VL-Plus 出 bbox，SAM2 出粗 mask。

解释：这里 SAM2 只是数据构造工具，不是最终模型推理依赖。

### D14

粗 mask 用作后续合并 SA-1B 碎片的 prior。

传统 CV 对照：先得到一个目标区域参考，再从碎片 mask 中挑相关部分。

### D15

细 mask 合并阶段会丢弃小碎片和不相交碎片。

解释：减少噪声。

### D16

论文用 IoF，而不是 IoU，判断小 fragment 是否属于目标。

传统 CV 对照：如果 fragment 很小，IoU 可能低，但 fragment 大部分在目标内就应保留。

### D17

合并后做填洞、形态学清理、连通域过滤。

解释：这是传统 CV 很熟悉的 mask 后处理。

### D18

如果清理导致面积大幅下降，就回退到 coarse mask。

工程意义：防止后处理把目标删坏。

### D19

MLLM 验证阶段检查类别一致性、覆盖完整性、污染、碎片和漏标。

解释：用强 MLLM 当自动质检员。

### D20

轻微边界误差可以接受，严重语义错或明显漏目标会被拒绝。

传统 CV 对照：训练数据不要求像素完美，但不能把别的物体当目标。

### D21

最后生成 referring caption。

解释：把 mask、label、box 转成“图像 + 指令 + mask”的训练样本。

## 5.3 SA1B-DeRS

### D22

DeRS 解决同类多实例需要属性/关系区分的问题。

例子：不是“person”，而是“右边骑车的人”。

### D23

每个样本用原图、目标 overlay 和 bbox 交给 Qwen3-VL-Plus 生成描述。

解释：让 MLLM 看目标高亮图，写出能唯一指代目标的短语。

### D24

描述从类别、属性、状态、相对位置、上下文关系生成。

传统 CV 对照：这是把视觉属性和关系转成语言监督。

### D25

再把生成的描述喂回 Qwen3-VL-Plus，让它重新定位。

解释：这相当于自检：描述是否真的能指回原目标。

### D26

如果重新定位 bbox 与 GT bbox IoU 低于 0.8，就丢弃。

工程意义：过滤歧义描述。

### D27

最后按面积比例做 saliency filtering。

传统 CV 对照：太小的碎片目标训练价值低、噪声高，先过滤掉。

## 5.4 数据分布和 ORS-Bench

### D28

论文统计 SA1B-ORS 规模、类别数、长尾分布和指令类型。

解释：开放世界数据必须看长尾，不只看 top 类别。

### D29

描述型指令占比较高，用来增强复杂语言理解。

工程意义：如果训练数据都是类别名，模型遇到长描述就会弱。

### D30

词云显示描述中有大量方位词和属性词。

传统 CV 对照：这些词就是空间关系和外观属性 supervision。

### D31

ORS-Bench 分 ID 和 OOD。

解释：ID 看同分布效果，OOD 看泛化边界。

### D32

ID benchmark 有四种指令：单实例类别、多实例类别、短语、描述。

这四种覆盖从简单到复杂的语言指代。

### D33

OOD benchmark 关注类别、面积、指令、遮挡、光照、风险场景六类 shift。

传统 CV 对照：这比普通 val/test 更像部署风险测试。

### D34

风险场景包括自动驾驶和医学。

解释：这些领域低容错，同时通常和通用训练集分布差异大。

## 6. Implementation 逐句解释

## 6.1 Multi-stage Training

### T01

论文采用两阶段训练，让模型同时获得分割能力和通用多模态能力。

解释：只训分割会变强分割但可能损害问答/OCR/推理。

### T02

Stage 1 目标是建立 referring segmentation 能力。

传统 CV 对照：先专门训练 dense prediction。

### T03

利用预训练 backbone 已有的 grounding 能力。

解释：Qwen3-VL 已会找目标框，不用从零学语言定位。

### T04

Stage 1 训练 LoRA、vision encoder 和 mask decoder。

工程意义：LLM 主体大权重冻结，只用 LoRA 适配；视觉 encoder 也稍微适配分割。

### T05

训练时模型输出结构化 bbox 和 mask placeholder。

解释：文本端仍生成 JSON，mask 由 decoder 根据 placeholder 对应 token 来预测。

### T06

mask decoder 把 mask token 映射到 binary mask。

核心：文本里的 `<mask_token>` 是语言输出和图像 mask 的桥。

### T07

Stage 2 目标是恢复和增强通用多模态能力，同时保留分割。

解释：避免模型变成只会分割的专用模型。

### T08

Stage 2 先合并 LoRA，再全量微调 LLM 和 mask decoder，冻结 vision encoder。

传统 CV 对照：第一阶段适配视觉特征，第二阶段稳定视觉、调语言推理。

### T09

Stage 2 数据混合分割、通用理解和推理。

解释：这就是 multi-task training，用数据比例控制能力平衡。

### T10

推理数据用原始 Qwen3-VL-Instruct 蒸馏。

解释：用 teacher 模型生成 STEM 推理数据，防止分割训练损害推理。

## 6.2 Prompt 和输出格式

### T11

论文遵循 Qwen3-VL 的 ChatML 对话格式。

解释：图像和文字被组织成 system/user/assistant 这样的聊天上下文。

### T12

指令分 category-based 和 description-based。

例子：

- category-based：找某类的所有实例。
- description-based：找符合描述的特定目标。

### T13

输出被约束成机器可解析 JSON。

工程意义：后续代码可以稳定 parse bbox、label 和 mask placeholder。

### T14

每个目标包含 bbox、label、mask placeholder 三个字段。

传统 CV 对照：bbox 是检测结果，mask placeholder 是给 mask decoder 的索引/桥接信号。

### T15

mask placeholder 不是真正的 mask 字符串。

解释：它是一个特殊 token 序列，告诉模型“这里有一个需要 decoder 生成的 mask”。

## 6.3 训练超参

### T16

backbone 用 Qwen3-VL-4B。

解释：4B 是语言-视觉主干，不是 mask decoder 参数。

### T17

新增 mask decoder 约 17M 参数。

传统 CV 对照：相对 4B 主体非常小。

### T18

Stage 1 LoRA rank 为 32，训练 10000 iterations。

解释：LoRA 是低秩适配，不直接训练完整 LLM 权重。

### T19

Stage 1 学习率是 `1e-4`，vision encoder 用 `0.01x`。

工程意义：mask decoder/LoRA 学快一点，视觉 encoder 小心微调，避免破坏预训练表征。

### T20

loss 是文本生成 loss 加分割 loss。

解释：既要能生成正确 JSON/bbox，又要 mask 对。

### T21

分割 loss 是 BCE + Dice。

传统 CV 对照：BCE 管逐像素分类，Dice 管区域重叠和前景比例。

### T22

Stage 2 学习率很小，为 `7e-7`。

解释：全量 LLM 微调风险大，所以用保守学习率。

## 9. 从传统 CV 角度重新理解整篇论文

### 9.1 它不是普通实例分割

普通实例分割：

```text
image -> masks for predefined classes
```

Qwen3-VL-Seg：

```text
image + language -> bbox JSON + mask for the referred target
```

关键差异是语言输入。类别不是固定 head 中的 class id，而是 LLM token 中的语义。

### 9.2 它也不是简单的 “Qwen + SAM”

Qwen + SAM：

```text
Qwen predicts box
SAM uses box prompt to segment
```

Qwen3-VL-Seg：

```text
Qwen features + mask token + bbox + image
lightweight decoder predicts mask
```

区别：

- 不依赖外部 SAM。
- mask decoder 与 Qwen 特征联动。
- bbox 是 decoder 内部结构先验，不只是给另一个模型的 prompt。

### 9.3 为什么 bbox 很重要

如果只给语言，mask decoder 要在整图里找目标，难度大。

如果给 bbox，问题变成：

```text
在这个大致区域内，根据语言语义和图像细节切出目标边界
```

这对小 decoder 友好很多。

### 9.4 为什么还要语言特征

bbox 内可能有多个物体或部件。语言特征告诉模型分哪个：

- “人” vs “人的帽子”
- “车” vs “车顶灯”
- “左边的杯子” vs “右边的杯子”

传统分割只靠类别 id；VL 分割靠整句语义。

### 9.5 为什么高分辨率图像分支重要

MLLM vision token 通常低分辨率。直接上采样容易：

- 边界糊
- 细长结构丢
- 小目标漏

浅层图像分支补的是传统 CV 很熟悉的边缘、纹理、局部颜色。

### 9.6 为什么 P@0.9 重要

Dice/mIoU 高不代表边界完美。P@0.9 要求 IoU 很高，能更敏感地反映：

- 边界是否贴合
- 是否欠分割
- 是否过分割
- 是否漏掉目标细部

论文消融里高分辨率和多尺度模块主要体现在 P@0.9。

## 10. 对当前舌体分割仓库的启发

当前仓库任务是单类舌体二值分割，和论文完整 open-world RES 不同。

最该复用的思想：

- bbox 作为结构先验。
- bbox soft gate，扩大 15%，alpha 约 20。
- 高分辨率 RGB shallow branch。
- 第一轮 mask 后做 refinement。
- 指标补 P@0.9 / cIoU，观察边界质量。

暂时不必复刻的内容：

- SA1B-ORS 数据构造。
- 多实例 category 指令。
- description-based open-world 训练。
- Stage 2 通用多模态混合训练。
- 全量 Qwen3-VL fine-tune。
- 完整 transformer query decoder。

当前仓库最关键的下一步不是继续堆复杂 decoder，而是把 bbox 来源拆开评估：

```text
GT bbox -> mask：验证 mask head 上限
jitter bbox -> mask：验证 bbox 鲁棒性
Qwen generated bbox -> mask：验证端到端能力
```

如果 generated bbox 不准，mask decoder 再强也会受限。

## 11. 一页速记

Qwen3-VL-Seg 解决的问题：

```text
语言指代目标 -> 精确 mask
```

它为什么需要 Qwen3-VL：

```text
Qwen3-VL 懂语言，能开放世界找 bbox
```

它为什么还需要 mask decoder：

```text
bbox 太粗，不能表达像素边界
```

它为什么不用 SAM：

```text
减少外部大模型依赖，让系统更一体化
```

decoder 四件事：

```text
多尺度视觉特征 -> 保空间
box + mask token query -> 定目标
box-gated high-res fusion -> 补边界
mask-aware refinement -> 二次修正
```

训练两阶段：

```text
Stage 1：专门学分割
Stage 2：混合数据恢复通用多模态能力
```

传统 CV 人最容易理解的类比：

```text
Qwen3-VL-Seg = open-vocabulary detector + language-conditioned Mask2Former/U-Net-like decoder
```

但它的关键不是检测器类别头，而是自然语言和 MLLM token。
